# Web research

Produce a trustworthy, cited answer from web sources — not a guess. Depth scales
with the stakes: a quick fact needs one good source; a decision needs several
independent ones.

## Plan

1. Restate the question and pin down what "answered" means (a number? a comparison?
   a how-to?). Note the timeframe — is recency critical?
2. Break it into sub-questions and draft 2–5 distinct search queries. Vary phrasing;
   don't just re-run one query.

## Gather

- Fan out searches, then **open the actual sources** — don't rely on snippets.
- Prefer primary and authoritative sources: official docs, standards, the vendor,
  original research, first-hand reporting. Treat forums/blogs/AI summaries as leads
  to verify, not as the answer.
- For anything time-sensitive, check the publication date and look for the most
  recent authoritative statement.

## Verify (the load-bearing step)

- Corroborate each key claim across **≥2 independent** sources. Independent means
  not all citing the same origin.
- Actively look for disconfirming evidence and note disagreement between sources
  instead of averaging it away.
- Distinguish fact from opinion/projection. Watch for outdated pages, marketing,
  and confident-but-unsourced claims.

## Synthesize

- Answer the question directly and first. Then the supporting detail.
- Cite sources inline for every non-obvious claim (title + link). If sources
  conflict, say so and explain which you trust and why.
- Separate what you know from what you inferred. State remaining uncertainty and
  what would resolve it.
- Don't fabricate citations or quote a source you didn't open.
